# substrate-guard v13.2.2 — Bandit policy + code hardening

## Summary

- **`bandit.yaml`** at repo root: excludes `tests` / `benchmarks` / build dirs; skips **B404** / **B603** (intentional `subprocess` without shell).
- **`bandit>=1.7`** added to **`[project.optional-dependencies] dev`**.
- **`tests/test_bandit_policy.py`**: runs Bandit with that config on `substrate_guard/`; failures block CI like any other test.

## Code quality / security hygiene

- **`audit.py`:** PostgreSQL queries are **static module-level strings** with bound parameters (no f-string SQL); table counts use a fixed **`_COUNT_QUERIES`** map.
- **`offline/local_store.py`:** parameterized `IN` clause; **`# nosec B608`** on the execute line (placeholders are `?` only; ids bound as parameters).
- **`observe/tracer.py`:** `except queue.Full` instead of bare `except` when the bounded queue is full.
- **`offline/sync.py`:** `contextlib.suppress` instead of empty `except` on cursor/connection close.
- **`attest/device_key.py`**, **`attest/local_ca.py`:** `assert` replaced with **`RuntimeError`** / guards where appropriate for predictable behavior under `-O`.

## Compatibility

No intentional API breaks; attest behavior is stricter on impossible states (explicit errors instead of `AssertionError`).
