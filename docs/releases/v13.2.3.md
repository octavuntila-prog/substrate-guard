# substrate-guard v13.2.3 — LocalStore `mark_synced` without dynamic SQL

**`LocalStore.mark_synced`** now uses **`sqlite3.Connection.executemany`** with a static statement `UPDATE events SET synced = 1 WHERE id = ?` instead of a dynamically sized `IN (...)` clause.

This removes the need for **`# nosec B608`**, eliminates the Bandit “nosec without matching test” warning, and keeps ids strictly bound as parameters.

No API change; behavior is equivalent (duplicate ids in the input list still apply multiple updates harmlessly).
