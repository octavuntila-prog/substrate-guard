# substrate-guard v13.2.4 — Operational clarity + `doctor`

## Goals

Make it obvious **what works out of the box** vs **what needs Linux/kernel/OPA/Postgres**, and remove broken Quick Start commands.

## Added

- **`substrate-guard doctor`** — reports Python, `z3-solver`, `opa`, `bcc`, `psycopg2`, `cryptography`, and whether `.rego` files exist under `policy/policies/`.
- **`docs/FUNCTIONAL_ROADMAP.md`** — tiers A–D (mock pipeline, eBPF, OPA, Postgres audit).
- **`requirements.txt`** / **`requirements-dev.txt`** — thin `-e .` installs aligned with `pyproject.toml`.

## Behavior

- **Tracer:** on **Windows**, skip eBPF init immediately and use **mock tracer** with an explicit log line (instead of relying on generic import failures).

## Docs

- **README** Quick Start rewritten to match real entrypoints (`python -m substrate_guard.cli`, `pip install -e ".[dev]"`).
- **REPRODUCING.md** links to `doctor` and the roadmap.

## Tests

- **`tests/test_cli_doctor.py`** — smoke + `--json` reserved exit code.
