# v13.3.2 — Patch Release (Sentinel-Aware Policy Logging)

**Released:** 2026-05-01

## Summary

Distinguishes by-design built-in policy fallback (sentinel
`BUILTIN_POLICY_PATH`) from genuine policy path misconfiguration in log
output. The previous WARNING fired on every cron run, including when the
system was operating exactly as designed — masking genuine
misconfiguration when a user-specified path was missing.

## Changes

### Bug Fix — Policy Engine Logging

**Before:**

```
WARNING substrate_guard.policy: Policy path __builtin_fallback__ not found.
Using built-in rules only.
```

This fired in every default-mode cron run, suggesting an error condition
where none existed. A genuine policy path typo (e.g.,
`--policy /etc/missing.rego`) would produce the same WARNING text,
making it indistinguishable from default operation.

**After:**

For sentinel `BUILTIN_POLICY_PATH` (default operation):

```
INFO substrate_guard.policy: No Rego policy configured. Using built-in Python rules.
```

For a user-specified path that doesn't exist (genuine misconfiguration):

```
WARNING substrate_guard.policy: Policy path /etc/missing.rego not found.
Falling back to built-in Python rules.
```

### Implementation

- Added import of `BUILTIN_POLICY_PATH` from `substrate_guard.constants`
  to `policy/engine.py`
- Sentinel-aware dispatch in `_load_policies()` else branch
- INFO level for sentinel case (default operation, not warning-worthy)
- WARNING level preserved for genuine missing-path case
