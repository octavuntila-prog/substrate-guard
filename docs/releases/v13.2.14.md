# substrate-guard v13.2.14 — reproducibility fixes

Three grouped fixes addressing a silent-broken install discovered during
AST benchmark validation against v13.2.13.

## Changes

- **fix(deps)**: Promoted `tree-sitter>=0.21,<0.26` and `tree-sitter-bash>=0.21,<0.26`
  from `[dev]` extras to core `dependencies`. `substrate_guard.ast_parse.parser`
  imports both at runtime and the `ast_bash_*` structural rules are production
  features, not dev tooling. Before this change, a plain `pip install
  substrate-guard` produced a working-looking install where `structural_scan()`
  returned zero violations on every bash payload with no indication to the user.
  Upper bound `<0.26` covers the empirically tested 0.21–0.25 range.
- **fix(ast_parse)**: `tree_sitter_bash_available()` now emits a single
  `ImportWarning` on first call when the packages are unavailable (cached via
  module-level state, matching the existing `_parsers` idiom). No more silent
  skip. Users see exactly what to install.
- **fix(version)**: `substrate_guard.__version__` synced to `"13.2.14"`.
  In v13.2.13 the dunder was stale at `"13.2.12"` while `pyproject.toml` read
  `"13.2.13"`; `verify_live.py --version` auto-detect would have reported
  the wrong label.

No API changes, no policy changes, no schema changes. The AST benchmark
corpus `observed.version` label advances from `v13.2.13` to `v13.2.14`
but detection behaviour is identical (5 detection divergences on the
47-sample corpus, the same Class A/B/C sampled in the catalog).

## Note for paper #5 readers

The AST benchmark corpus measurements published in *The AI Black Box:
Six-Layer Verification Architecture* §3.2.1 (Zenodo DOI
10.5281/zenodo.19334382, v3) were computed against substrate-guard with
`tree-sitter-bash` installed. With v13.2.13 and earlier, reviewers
reproducing those numbers from a fresh `pip install substrate-guard`
would have obtained silent-skip behaviour on the 21 bash payloads,
producing ~19 apparent detection divergences instead of the 5
documented. The apparent AST detection rate would have collapsed from
the reported 69.2% to roughly 33% — a reproducibility failure entirely
attributable to the packaging bug, not to the tool's structural rules.

Starting with v13.2.14, `pip install substrate-guard` is sufficient to
reproduce the paper numbers exactly. For strict version-pinned
reproducibility:

    pip install substrate-guard==13.2.14

The AST benchmark corpus (Zenodo DOI TBD) can then be re-run with:

    python scripts/verify_live.py --version v13.2.14

Expected output: 47 samples evaluated, 0 errors, 5 detection divergences,
0 rule-name divergences.
