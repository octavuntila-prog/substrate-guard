# substrate-guard v13.2.6 — stack_audit hardening + CI docker-stack-audit

## Scripts

- **`scripts/stack_audit.sh`** — explicit success/failure paths: wait **120s** for `pg_isready`, propagate **audit exit code**, **`teardown`** (`docker compose down`) on success and on failure unless **`SKIP_CLEANUP=1`**.
- **`scripts/stack_audit.ps1`** — same semantics; **`$env:SKIP_CLEANUP="1"`** to keep containers.

## CI

- **`.github/workflows/docker-stack-audit.yml`** — runs `./scripts/stack_audit.sh` on **Ubuntu** (`workflow_dispatch` + **weekly cron**). Not on every push (image build includes full `pytest`).
