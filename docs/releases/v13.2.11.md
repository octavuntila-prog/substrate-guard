# substrate-guard v13.2.11 — PyYAML required for YAML structural scan

**`pyyaml>=6.0`** moved from optional **dev** to **core** `dependencies` in `pyproject.toml`, so `pip install substrate-guard` (or `pip install -e .`) always includes YAML structural checks in `structural_scan` / `verify_cli`.

`yaml_structural_issues` now imports `yaml` at module load (no silent empty fallback when the package is missing).
