# substrate-guard v13.2.1 — Open-source hardening

Patch release focused on **supply chain**, **disclosure policy**, and one **CLI security** fix.

## Changes

- **`substrate_guard.cli`:** tool benchmark no longer uses `os.system` / shell — runs `run_tool_benchmark.py` with `subprocess` + `sys.executable` (addresses Bandit B605).
- **Dependencies:** `cryptography>=46.0.6` (addresses known CVEs in older releases).
- **`SECURITY.md`:** how to report vulnerabilities, scope, and pointer to CI checks.
- **GitHub:** Dependabot (pip + Actions), workflow **`supply-chain`** (`pip>=25.3` then `pip-audit`), workflow **CodeQL** (Python).
- **Docs:** README / REPRODUCING / AUDIT_COMPLEX updated; `.gitignore` includes `.venv-audit/`.

## Compatibility

Backward compatible with v13.2.0 for API and CLI behavior (benchmark output unchanged aside from implementation).
