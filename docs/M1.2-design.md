## 1. Executive Summary

### Goal

Transform substrate-guard from hardcoded Python built-in policy dispatch to **user-configurable** engine selection with conservative defaults. Enable both Rego (via OPA) and Python built-in policy engines at runtime without code changes or redeployment.

### Non-goals (explicitly excluded)

- **Changing Guard class API** — `Guard.__init__(policy=...)` signature unchanged
- **Auto-detect Rego/builtin** — no progressive fallback logic (conservative default)
- **Dual-mode evaluation** — no simultaneous Rego + built-in evaluation
- **Event replay from JSON** — no `--replay` flag (deferred to potential v13.4)
- **Rego policy updates** — `agent_safety.rego` untouched in this milestone
- **OPA install automation** — assumed already done via M1.1

### Success criteria

- `python3 -m substrate_guard.audit --policy rego` produces valid audit JSON with Rego dispatch
- `python3 -m substrate_guard.audit --policy builtin` identical to current v13.2.16 behavior
- `python3 -m substrate_guard.audit` (no flag) identical to current v13.2.16 (zero regresie)
- Env var `SUBSTRATE_GUARD_POLICY=rego` activates Rego without CLI flag
- Precedence: CLI flag > env var > hardcoded default
- Cron-audit.sh unchanged in initial M1.2 ship (production cron identical post-upgrade)
- All existing tests pass + new tests cover both modes
- Audit JSON schema includes `policy_engine: "rego" | "builtin"` field at top level

## 2. Architecture Decisions

### D1 — CLI Syntax: `--policy <mode>`

**Decision:** Single flag with enum value.

```bash
python3 -m substrate_guard.audit --policy rego
python3 -m substrate_guard.audit --policy builtin
python3 -m substrate_guard.audit                    # default = builtin
```

**Rationale:**
- Extensible (future `--policy hybrid`, `--policy strict`, `--policy async`)
- Industry standard (gcc `-O0/-O1`, docker `--log-driver`, ansible `--become-method`)
- Argparse auto-validation via `choices=['rego', 'builtin']` — typo detection built-in
- Self-documenting in logs and cron scripts

**Rejected alternatives:**
- `--use-rego` / `--use-builtin` boolean flags: not extensible, mutually-exclusive ambiguity
- `--policy-path <dir>` auto-detect: semantic surprise, doesn't express intent

### D2 — Default Behavior: Conservative + Env Var Hybrid

**Decision:** Precedence chain with safe default.

```
Resolution order (first match wins):
1. CLI flag --policy
2. Environment variable SUBSTRATE_GUARD_POLICY
3. Hardcoded default: "builtin"
```

**Implementation:**

```python
def resolve_policy_mode(args) -> tuple[str, str]:
    """Returns (mode, source) — source is one of 'cli', 'env', 'default'."""
    VALID_MODES = {'rego', 'builtin'}

    # CLI flag has highest precedence
    if args.policy:
        if args.policy not in VALID_MODES:
            raise ValueError(f"Invalid --policy value: {args.policy}. Expected: {VALID_MODES}")
        return args.policy, 'cli'

    # Env var second
    env_value = os.environ.get('SUBSTRATE_GUARD_POLICY')
    if env_value:
        if env_value not in VALID_MODES:
            logger.warning(f"Invalid SUBSTRATE_GUARD_POLICY={env_value}; ignoring. Expected: {VALID_MODES}")
        else:
            return env_value, 'env'

    # Hardcoded default
    return 'builtin', 'default'
```

**Rationale:**
- Zero regresie on current cron (default = builtin, identical to v13.2.16)
- Paper v3 §5.3.1 remains accurate (production state unchanged by default)
- Env var allows per-server configuration without CLI changes
- CLI override for ad-hoc testing and dev environments
- Invalid env var doesn't crash — warns and falls back (resilient)

**Rejected alternatives:**
- Progressive auto-detect: silent activation risk on cron (Research Agency has OPA installed since M1.1)
- Env var pure (no CLI): less flexible for testing
- Conservative pure (no env): requires cron-audit.sh changes for production activation

### D3 — `policy="nonexistent/"` Fate: BUILTIN_POLICY_PATH Constant

**Decision:** Rename magic string to documented constant.

```python
# In substrate_guard/constants.py (new file) or at top of audit.py
BUILTIN_POLICY_PATH = "__builtin_fallback__"
"""
Sentinel path that intentionally does not resolve to any filesystem location.
When passed to PolicyEngine(policy_path=BUILTIN_POLICY_PATH), causes
self._policies = [], which dispatches to the Python built-in fallback engine.

This is NOT a typo. This is intentional: Layer 2 remains active (evaluating
events via Python built-in rules), but Rego loading is bypassed.
"""
```

**Mapping:**

```python
def resolve_policy_path(mode: str) -> str:
    """Map policy mode to path suitable for Guard(policy=<path>)."""
    if mode == "rego":
        # Path to shipped Rego policies directory
        return str(Path(__file__).parent / "policy" / "policies")
    elif mode == "builtin":
        return BUILTIN_POLICY_PATH
    raise ValueError(f"Unknown policy mode: {mode}")
```

**Rationale:**
- Zero runtime behavior change vs current "nonexistent/" (both non-existent paths trigger `self._policies = []`)
- Self-documenting intent: grep for `BUILTIN_POLICY_PATH` finds deliberate design, not "what typo is this?"
- Paper v3 §5.3.1 can cite constant by name, making claim audit-traceable
- Backwards compatible: `Guard.__init__()` signature unchanged

**Rejected alternatives:**
- Remove `policy=` kwarg (policy=None): breaking change, Layer 2 semantically different (skip vs active), invalidates paper v3 production claims
- Hardcode valid path (progressive default): contradicts D2

### D4 — Retroactive Audit: Two Runs + Deterministic Window

**Decision:** Simple — two separate audit runs, same time window via new CLI flags.

**New CLI flags:**

```bash
# Deterministic window (both flags optional; default = --hours 24)
--from YYYY-MM-DD        # inclusive start date
--to YYYY-MM-DD          # inclusive end date
--hours N                # backwards-compat, alternative to --from/--to

# Behavior:
# - If --from and --to set: audit events in [from_date, to_date] range
# - If only --hours N set: audit events in last N hours (current behavior)
# - If all three specified: --from/--to wins, --hours ignored with warning
```

**M1.3 usage pattern:**

```bash
# Fixed window, independent of clock time
WINDOW_START="2026-04-01"
WINDOW_END="2026-05-15"

# Run 1: builtin baseline
python3 -m substrate_guard.audit \
    --from $WINDOW_START --to $WINDOW_END \
    --policy builtin \
    --output /tmp/builtin_42day.json

# Run 2: rego evaluation on same events
python3 -m substrate_guard.audit \
    --from $WINDOW_START --to $WINDOW_END \
    --policy rego \
    --output /tmp/rego_42day.json

# Compare (separate script, scope of M1.3 not M1.2)
python3 scripts/compare_policy_runs.py /tmp/builtin_42day.json /tmp/rego_42day.json
```

**Rationale:**
- Simplest implementation (~20 LOC for window flags, zero changes to evaluation logic)
- Deterministic: same DB query twice = same events = comparable results
- Debuggable: each run independent, failure in one doesn't affect other
- No JSON schema breakage: `evaluation` dict remains singular per run
- Comparison script is separate concern (M1.3 deliverable, not M1.2)

**Rejected alternatives:**
- Dual-mode run: scope creep, JSON schema break, harder debugging
- Event replay mode: ~150 LOC extra, overscoped for M1.2 target

## 3. API Changes

### 3.1 New CLI Flags

```
--policy {rego,builtin}        Policy engine to use (default: builtin)
--from YYYY-MM-DD              Inclusive start date for event window
--to YYYY-MM-DD                Inclusive end date for event window
```

Existing flags unchanged: `--hours`, `--output`, etc.

### 3.2 New Environment Variables

```
SUBSTRATE_GUARD_POLICY={rego|builtin}
    Default policy engine if --policy flag not provided.
    Ignored if invalid value (logs warning).
```

### 3.3 New JSON Schema Field

Audit report JSON adds one top-level field:

```json
{
  "timestamp": "2026-04-22T04:00:02.643412",
  "substrate_guard_version": "13.2.16",
  "policy_engine": "builtin",                     // NEW
  "policy_engine_source": "default",              // NEW (cli | env | default)
  "db_records": {...},
  "events_generated": 58,
  "evaluation": {...},
  "layers": {
    "observe": "mock",
    "policy": "builtin (7 rules)",                // unchanged semantics
    "verify": "z3"
  }
}
```

**Rationale:**
- `policy_engine` is machine-readable (enum), `layers.policy` is human-readable string
- `policy_engine_source` enables debugging: "why did this run use rego?" → "env var set it"
- Consumers that parse `d['evaluation']['violations']` continue working (no schema break)

### 3.4 Startup Logging

New log line at audit.py startup:

```
INFO Policy engine: builtin (source: default, path: __builtin_fallback__)
```

OR:

```
INFO Policy engine: rego (source: cli, path: /opt/substrate-guard/substrate_guard/policy/policies)
```

## 5. Testing Strategy

### 5.1 New test file: `tests/test_policy_mode_resolution.py`

```python
"""Tests for M1.2 policy mode resolution."""

import os
import pytest
from argparse import Namespace
from substrate_guard.audit import resolve_policy_mode, resolve_policy_path
from substrate_guard.constants import BUILTIN_POLICY_PATH

class TestPolicyModeResolution:
    def test_cli_flag_wins_over_env(self, monkeypatch):
        monkeypatch.setenv('SUBSTRATE_GUARD_POLICY', 'rego')
        args = Namespace(policy='builtin')
        mode, source = resolve_policy_mode(args)
        assert mode == 'builtin'
        assert source == 'cli'

    def test_env_var_used_when_cli_absent(self, monkeypatch):
        monkeypatch.setenv('SUBSTRATE_GUARD_POLICY', 'rego')
        args = Namespace(policy=None)
        mode, source = resolve_policy_mode(args)
        assert mode == 'rego'
        assert source == 'env'

    def test_default_when_neither_set(self, monkeypatch):
        monkeypatch.delenv('SUBSTRATE_GUARD_POLICY', raising=False)
        args = Namespace(policy=None)
        mode, source = resolve_policy_mode(args)
        assert mode == 'builtin'
        assert source == 'default'

    def test_invalid_env_falls_back_to_default(self, monkeypatch, caplog):
        monkeypatch.setenv('SUBSTRATE_GUARD_POLICY', 'invalid_value')
        args = Namespace(policy=None)
        mode, source = resolve_policy_mode(args)
        assert mode == 'builtin'
        assert source == 'default'
        assert 'Invalid SUBSTRATE_GUARD_POLICY' in caplog.text

    def test_invalid_cli_raises(self):
        args = Namespace(policy='invalid_value')
        with pytest.raises(ValueError, match='Invalid --policy'):
            resolve_policy_mode(args)


class TestPolicyPathResolution:
    def test_builtin_maps_to_sentinel(self):
        assert resolve_policy_path('builtin') == BUILTIN_POLICY_PATH

    def test_rego_maps_to_real_path(self):
        path = resolve_policy_path('rego')
        assert path.endswith('policy/policies') or path.endswith('policy\\policies')

    def test_unknown_mode_raises(self):
        with pytest.raises(ValueError, match='Unknown policy mode'):
            resolve_policy_path('unknown_mode')
```

## 6. Migration Path

### v13.2.16 → v13.3.0

**Backwards compatibility guarantee:**

- User running `python3 -m substrate_guard.audit` (no flags) gets **identical output** to v13.2.16, with one addition: `policy_engine: "builtin"` field.
- All existing JSON consumers continue working (no field removal, no type changes).
- Cron-audit.sh requires **zero changes** — runs with default builtin automatically.
- Environment without `SUBSTRATE_GUARD_POLICY` set → default builtin.

**Upgrade sequence:**

```bash
# On production server (Research Agency or similar):
cd /opt/substrate-guard
git fetch origin --tags
git checkout v13.3.0

# No reinstall needed (PYTHONPATH-based import)
# Runtime version auto-updates

# Verify:
python3 -c "import substrate_guard; print(substrate_guard.__version__)"
# Expected: 13.3.0

# Test dry-run with default mode:
python3 -m substrate_guard.audit --hours 1 --output /tmp/test.json
# Verify JSON has policy_engine: "builtin"

# Cron continues working identically. Next scheduled run produces v13.3.0 output.
```

### Activation of Rego (operational decision, not part of M1.2)

When ready to activate Rego in production:

**Option 1 — Per-run CLI:**
```bash
python3 -m substrate_guard.audit --policy rego ...
```

**Option 2 — Global via env var:**
```bash
# In cron-audit.sh, add:
export SUBSTRATE_GUARD_POLICY=rego
```

**Option 3 — Edit cron directly:**
```bash
# crontab -e:
0 4 * * * /usr/local/bin/cron-with-alert substrate-guard-audit \
    bash -c 'export SUBSTRATE_GUARD_POLICY=rego && /opt/substrate-guard/scripts/cron-audit.sh'
```

**Recommended:** Option 2 (env var in cron-audit.sh). Clear, auditable, single source of truth per server.
